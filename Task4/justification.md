## Обоснование конфигурации инфраструктуры

### Infrastructure as Code (IaC)

Вся инфраструктура описана в виде кода, что даёт следующие преимущества:
- Воспроизводимость: любой разработчик может развернуть идентичную среду локально или в другом окружении.
- Масштабируемость: IaC-инструменты очень полезны при масштабировании решений, где нужны специальные настройки окружения при развёртывании, чтобы добиться эталонной конфигурации на множество экземпляров одновременно.
- Аудит и управление изменениями: вся история изменений фиксируется в системе контроля версий, что упрощает откат, анализ и совместную работу.

### Terraform и декларативный подход

В данном проекте развертывание виртуальной инфраструктуры полностью автоматизировано с использованием Terraform.
Такой подход обеспечивает предсказуемость, воспроизводимость и упрощает управление средой разработки и тестирования.

В декларативном подходе описывается конфигурация целевого состояния инфраструктуры или окружения, которое требуется, но при этом не указывается последовательность исполнения такой настройки. Простота — главное преимущество этого подхода. Не нужно описывать шаги для получения нужной конфигурации, а значит, уменьшается и риск сделать ошибку в написании настройки.

Terraform позволяет описать виртуальную инфраструктуру любой сложности и управлять ею с помощью собственного декларативного языка HCL (HashiCorp Configuration Language).

### Выбор ресурсов

- VM (yandex_compute_instance) — требуется для развёртывания тестового окружения.

- Disk (yandex_compute_disk) — отдельный ресурс, позволяющий гибко менять размер и характеристики диска.

- Network (default VPC + default subnet) — используется существующая сеть YC, так как задание не требует создания собственной.

### Настройки

- 2 vCPU, 2 GB RAM — минимальный рабочий размер для тестирования.

- NAT = true — нужен доступ в интернет для обновлений.

- SSH ключ — обеспечивает безопасный доступ по SSH.
