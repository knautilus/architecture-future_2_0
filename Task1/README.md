### Проблемы и затрагиваемые ими бизнес-сценарии

#### 1. Монолитное DWH (SQL Server 2008)

Все данные и бизнес-логика сосредоточены в одной устаревшей системе. Это "единая точка отказа", создающая узкое горлышко для всех процессов. Клиентский интерфейс и аналитические сервисы работают с одной БД, конкурируя за ресурсы.

На что влияет: страдает производительность, есть задержки в отчетности, сложность внесения изменений в БД из-за возможности сломать связанные системы, низкая масштабируемость, высокие риски простоя.

#### 2. Устаревшая технологическая база

SQL Server 2008 не поддерживается Microsoft, представляет угрозу безопасности. PowerBuilder и Camel — не самые популярные инструменты с малым сообществом.

На что влияет: страдает безопасность - риск утечки медицинских и финансовых данных; сложность и дороговизна разработки из-за отсутствия поддержки производителя; сложно находить разработчиков.

#### 3. Отсутствие доменных границ

Все системы сильно связаны через общее DWH. Изменение в финтехе может сломать логику в медицинской отчетности.

На что влияет: cтрадает независимое развитие, невозможно развивать направления отдельно, высокая стоимость изменений.

#### 4. Низкая производительность аналитики

Множество трансформаций в DWH, большие объемы данных, сложные запросы выполняются часами. Отсутствует портал самообслуживания, пользователи ждут ручных выгрузок.

На что влияет: pатрагивает принятие решений; менеджеры получают отчеты с задержкой, что приводит к запаздывающим решениям.

#### 5. Отсутствие разграничения доступа

Нет IAM/SSO, доступы не разграничены. Нет аудита доступа к данным.

На что влияет: cуществует риск утечки чувствительных данных; может повлечь за собой нарушение 152-ФЗ, юридические и репутационные риски.

### Приоритезация проблем

Используем метод приоритезации MoSCoW: Must have, Should have, Could have, Won't have

| Приоритет   | Проблема                             | Почему важно                                               |
| ----------- |------------------------------------- | ---------------------------------------------------------- |
| Must have   | Монолитное DWH                       | Затрудняет расширение бизнеса, страдает производительность |
| Must have   | Отсутствие разграничения доступа     | Риск утечек. Необходимо выполнять требования регуляторов   |
| Must have   | Отсутствие доменных границ           | Невозможно развивать направления отдельно                  |
| Should have | Низкая производительность аналитики  | Замедляет принятие решений                                 |
| Could have  | Устаревшая технологическая база      | Сложность и дороговизна разработки. Но это не блокирует цели компании на год |
| Won't have  | Одномоментный отказ от легаси-систем | Легаси-системы будут выводиться из эксплуатации постепенно, иначе могут возникнуть риски потери данных, простоя после релиза из-за ошибок в новых сервисах |
