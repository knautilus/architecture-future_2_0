### Проблемы и затрагиваемые ими бизнес-сценарии

#### 1 Монолитное DWH (SQL Server 2008)

Все данные и бизнес-логика сосредоточены в одной устаревшей системе. Это "единая точка отказа", создающая узкое горлышко для всех процессов. Клиентский интерфейс и аналитические сервисы работают с одной БД, конкурируя за ресурсы.

Затрагивает все сценарии. Страдает производительность, есть задержки в отчетности, сложность разработки, низкая масштабируемость, высокие риски простоя.

#### 2 Устаревшая технологическая база

SQL Server 2008 не поддерживается Microsoft, представляет угрозу безопасности. PowerBuilder и Camel — не самые популярные инструменты с малым сообществом.

Страдает безопасность - риск утечки медицинских и финансовых данных. Сложность и дороговизна разработки из-за отсутствия поддержки производителя. Cложно находить разработчиков.

#### 3 Отсутствие доменных границ

Все системы сильно связаны через общее DWH. Изменение в финтехе может сломать логику в медицинской отчетности.

Страдает независимое развитие. Невозможно развивать направления отдельно. Высокая стоимость изменений.

#### 4 Низкая производительность аналитики

Множество трансформаций в DWH, большие объемы данных, сложные запросы выполняются часами. Отсутствует портал самообслуживания, пользователи ждут ручных выгрузок.

Затрагивает принятие решений. Менеджеры получают отчеты с задержкой, что приводит к запаздывающим решениям.

#### 5 Отсутствие разграничения доступа

Нет IAM/SSO, доступы не разграничены. Нет аудита доступа к данным.

Существует риск утечки чувствительных данных. Может повлечь за собой нарушение 152-ФЗ, юридические и репутационные риски.

### Приоритезация

Используем метод приоритезации MoSCoW (Must have, Should have, Could have, Won't have)

#### Must have

- Отказ от монолитного DWH, т.к. он затрудняет расширение бизнеса. Миграция с legacy DWH на распределенные хранилища данных

- Отсутствие разграничения и аудита доступа - блокирует расширение бизнеса, необходимо выполнять требования регуляторов

#### Should have

- Отсутствие разделение на домены - блокирует развитие системы, то есть достижения целей бизнеса

- Низкая производительность аналитики. Необходимо разработать витрину данных.

#### Could have

- Устаревший технологический стек - затрудняет расширение бизнеса
